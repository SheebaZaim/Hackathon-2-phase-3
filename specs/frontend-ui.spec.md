# Teachers Planning App - Frontend UI Specification

## Design Philosophy
Create a clean, professional, academic interface that supports teachers in their planning and administrative tasks.

## Visual Requirements
- **Color Palette**: Neutral, professional colors (blues, grays, whites)
- **Typography**: Clean, readable fonts (preferably sans-serif)
- **Layout**: Dashboard-style with organized sections
- **Icons**: Small to medium-sized, professional icons only
- **Spacing**: Ample white space for readability

## Explicit Requirements
- NO gigantic icons or oversized elements
- NO playful book/cartoon imagery
- NO childish or student-oriented visuals
- Professional academic appearance maintained throughout
- Mobile responsive design
- Accessibility-compliant (WCAG 2.1 AA standards)

## Component Requirements
- Cards for organizing information
- Tables for displaying student/class data
- Forms for data entry
- Navigation sidebar for app sections
- Header with user profile and logout
- Search/filter functionality for data

## Page Structure
1. **Login Page**: Secure authentication form
2. **Dashboard**: Overview of classes, upcoming tasks
3. **Classes Page**: Manage teacher's classes
4. **Students Page**: View and manage students in classes
5. **Results Page**: Enter and view student academic results
6. **Profile Page**: Teacher profile and settings

## Responsive Behavior
- Mobile: Single column layout with collapsible navigation
- Tablet: Two-column layout where appropriate
- Desktop: Multi-panel dashboard layout

## Forbidden Elements
- Oversized icons (>48px in standard contexts)
- Cartoon graphics or animations
- Student-focused imagery
- Unprofessional color schemes
- Cluttered layouts

## User Experience
- Intuitive navigation between sections
- Quick access to most-used features
- Clear indication of current location in app
- Consistent styling across all pages
- Fast loading times
- Clear error messaging
